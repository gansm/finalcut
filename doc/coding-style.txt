============
Coding style
============

Formatting
----------
* Wrap lines at 72 characters (80 chars max)
* Use 2-space indentation. Do not use tabs.
* Place a space after keywords: if, switch, while, do, for, and return.
* Put spaces inside parentheses for conditions: if ( a > 5 ).
* Use a blank line before and after for, if, switch, and while/do-while
  blocks.
* Add a space after each comma in parameter lists.
* Place the opening curly brace "{" on a new line.

Naming Conventions
------------------
* Class name: UpperCamelCase
* Function: lowerCamelCase
* Callback function: cb_lowerCamelCase (prefixed with cb_)
* Variable: lower_snake_case

Class Declaration Order
-----------------------
Organize class members by access level:

  * public:
  * protected:
  * private:

Within each section, group members in the following order:

  * Using-declarations (e.g., using std::string;)
  * Typedefs and Enums
  * Constants (static const data members)
  * Constructors
  * Destructor
  * Overloaded operators (=, +, -, +=, etc.)
  * Accessors/Getters (getXY)
  * Mutators/Setters (setXY)
  * Predicates/Booleans (isXY)
  * General methods (including static methods)
  * Event handlers
  * Callbacks
  * Data members (excluding static constants)
